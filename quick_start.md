# Quick Start (быстрый запуск проекта)

## 1. Общая информация

Проект можно запускать в любой среде разработки, поддерживающей Python. В качестве примера ниже описан запуск через **PyCharm**, так как это одна из самых популярных IDE.

При необходимости проект также можно запускать через:

* командную строку (terminal);
* Visual Studio Code;
* другие IDE и интерпретаторы Python.

---

## 2. Требования

* Python версии **3.10 или выше**
* Установленные библиотеки из `requirements.txt`

---

## 3. Запуск проекта в PyCharm

### Шаг 1. Открытие проекта

1. Запустите PyCharm
2. Выберите `File → Open`
3. Укажите папку с проектом

---

### Шаг 2. Настройка интерпретатора Python

1. Перейдите в `File → Settings → Python Interpreter`
2. Нажмите ⚙️ → `Add Interpreter`
3. Выберите **Virtualenv** (рекомендуется)
4. Укажите версию Python
5. Сохраните настройки

---

### Шаг 3. Установка библиотек

#### Через интерфейс PyCharm

1. Откройте файл `requirements.txt`
2. Нажмите кнопку **Install requirements**
3. Дождитесь окончания установки

#### Через встроенный терминал

1. Откройте терминал PyCharm
2. Выполните команду:

```bash
pip install -r requirements.txt
```

---

### Шаг 4. Запуск проекта

1. Найдите файл `main.py`
2. Нажмите правой кнопкой мыши → **Run 'main'**
3. В консоли появится сообщение о запуске бота

---

## 4. Запуск без PyCharm

Проект можно запустить и без PyCharm:

1. Перейдите в папку проекта через терминал
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите проект:

```bash
python main.py
```

---

## 5. Возможные проблемы

* **Ошибка импорта модулей** — библиотеки не установлены
* **Бот не запускается** — неверный токен
* **Ошибка базы данных** — неверный путь к файлу БД или отсуствие папки 'db'

---

## 6. Примечание

Проект написан с использованием асинхронного подхода, что позволяет эффективно обрабатывать несколько событий одновременно. Такая архитектура широко используется при разработке сетевых приложений и ботов.

Структура проекта построена таким образом, чтобы его можно было легко дополнять новыми функциями без изменения уже существующего кода.
